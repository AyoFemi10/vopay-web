## Overview

VOPayX enforces rate limits to ensure platform stability and fair usage for all integrators.

## Default Limits

| Tier | Requests/minute | Requests/hour | Burst |
|------|----------------|---------------|-------|
| Sandbox | 60 | 1,000 | 10 |
| Standard | 300 | 10,000 | 50 |
| Business | 1,000 | 50,000 | 100 |
| Enterprise | Custom | Custom | Custom |

## Rate Limit Headers

Every API response includes these headers:

```http
X-RateLimit-Limit: 300
X-RateLimit-Remaining: 247
X-RateLimit-Reset: 1751400000
Retry-After: 42
```

| Header | Description |
|--------|-------------|
| `X-RateLimit-Limit` | Total requests allowed per window |
| `X-RateLimit-Remaining` | Requests remaining in the current window |
| `X-RateLimit-Reset` | Unix timestamp when the window resets |
| `Retry-After` | Seconds to wait before retrying (only on 429) |

## Handling 429 Responses

When you exceed the rate limit, VOPayX returns:

```json
{
  "success": false,
  "error": {
    "code": "RATE_LIMIT_EXCEEDED",
    "message": "Too many requests. Please retry after 42 seconds.",
    "retryAfter": 42
  }
}
```

### Retry with Exponential Backoff

```typescript
async function requestWithRetry<T>(
  fn: () => Promise<T>,
  maxRetries = 3
): Promise<T> {
  for (let attempt = 0; attempt <= maxRetries; attempt++) {
    try {
      return await fn();
    } catch (error: any) {
      if (error.status === 429 && attempt < maxRetries) {
        const retryAfter = error.retryAfter ?? Math.pow(2, attempt);
        await new Promise(r => setTimeout(r, retryAfter * 1000));
        continue;
      }
      throw error;
    }
  }
  throw new Error('Max retries exceeded');
}
```

```python
import time
import random

def request_with_retry(fn, max_retries=3):
    for attempt in range(max_retries + 1):
        try:
            return fn()
        except RateLimitError as e:
            if attempt == max_retries:
                raise
            delay = e.retry_after or (2 ** attempt + random.random())
            time.sleep(delay)
```

## Endpoint-Specific Limits

Some sensitive endpoints have stricter limits regardless of tier:

| Endpoint | Limit |
|----------|-------|
| `POST /auth/login` | 10 req/min per IP |
| `POST /auth/verify-otp` | 5 req/min per user |
| `POST /transfers` | 30 req/min per account |
| `POST /payroll/run` | 5 req/hour per account |
